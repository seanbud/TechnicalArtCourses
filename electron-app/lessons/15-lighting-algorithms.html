Lesson 15: Lighting Algorithms

The Missing Link: From Dot Products to Shading

You know the dot product. You've used Shader Graph. But there's a gap: how do simple lighting equations actually work? And how does per-vertex data become per-pixel data?

This lesson bridges that gap—from raw math to practical shading models.

How Pixels Get Vertex Data: Interpolation

This is the first mystery to solve. Vertices have normals. But fragment shaders run per-pixel. How does the pixel "get" a normal?

TRIANGLE WITH 3 VERTICES
        
        V0 (normal pointing up-left)
        /\
       /  \
      /    \
     /  P   \      ← P is a pixel being shaded
    /________\
  V1          V2
(normal       (normal
up-right)     straight up)

When the rasterizer determines that pixel P is inside this triangle, it calculates barycentric coordinates—three weights (w0, w1, w2) that sum to 1, representing how close P is to each vertex.

// Barycentric interpolation (GPU does this automatically)
pixelNormal = w0 * V0.normal + w1 * V1.normal + w2 * V2.normal;
pixelUV     = w0 * V0.uv     + w1 * V1.uv     + w2 * V2.uv;
pixelColor  = w0 * V0.color  + w1 * V1.color  + w2 * V1.color;

Key insight: The GPU interpolates ALL vertex outputs across the triangle surface. Whatever your vertex shader outputs, the fragment shader receives an interpolated version based on pixel position.

⚠️ Interpolated normals need re-normalization! Interpolating between two unit vectors doesn't produce a unit vector. Most lighting calculations require normalize(interpolatedNormal) in the fragment shader.

Lambert Diffuse: The Foundation

Lambert is the simplest physically-motivated lighting model. It describes how matte surfaces scatter light equally in all directions.

The Core Equation

     LIGHT SOURCE
          ☀️
           \
            \ L (light direction, pointing TO light)
             \
              \
        ──────●────── SURFACE
              ↑
              N (surface normal)

diffuse = max(0, dot(N, L))

What the dot product tells us:

  • N · L = 1 → Light hits straight-on (brightest)
  • N · L = 0 → Light grazes the surface (no contribution)
  • N · L < 0 → Light is behind the surface (clamped to 0)

HLSL Implementation

// In fragment shader
float3 N = normalize(input.worldNormal);  // Re-normalize!
float3 L = normalize(_LightPosition - input.worldPos);  // Direction TO light

float NdotL = max(0, dot(N, L));  // Clamp negative values

float3 diffuse = _LightColor * NdotL * _AlbedoColor;
return float4(diffuse, 1.0);

Clarification on your question: It's N dot L (normal dot light direction), not normal dot tangent. The tangent is used for normal mapping, which we'll touch on later.

Half-Lambert: Valve's Softening Trick

Standard Lambert has harsh shadows—areas facing away from light go completely black. Valve (Half-Life 2) created Half-Lambert to soften this.

COMPARISON:

Standard Lambert:    NdotL ranges [0, 1]
                     ████████░░░░░░░░  (harsh cutoff)

Half-Lambert:        (NdotL * 0.5 + 0.5) ranges [0.5, 1]
                     ████████████░░░░  (softer falloff)

Implementation

float NdotL = dot(N, L);
float halfLambert = NdotL * 0.5 + 0.5;  // Remap from [-1,1] to [0,1]
halfLambert = halfLambert * halfLambert;  // Square for curve

float3 diffuse = _LightColor * halfLambert * _AlbedoColor;

Use case: Stylized games, characters that need to remain readable in shadow, anywhere you want softer lighting without adding fill lights.

Phong Specular: Shiny Highlights

Lambert only handles diffuse (matte) reflection. Phong adds specular—the shiny highlights you see on glossy surfaces.

The Reflection Vector

         VIEW
          👁️
           \
            \ V (view direction, pointing TO camera)
             \
    L         \
     \    R   /
      \   ↗  /
       \ /  /
────────●────────  SURFACE
        ↑
        N

R = reflect(-L, N)  // Mirror L around N
specular = pow(max(0, dot(R, V)), shininess)

The idea: Light reflects off the surface. If the view direction aligns with the reflection direction, you see a bright highlight. The pow() controls how tight/sharp the highlight is.

HLSL Implementation

float3 N = normalize(input.worldNormal);
float3 L = normalize(_LightPosition - input.worldPos);
float3 V = normalize(_CameraPosition - input.worldPos);  // TO camera

float3 R = reflect(-L, N);  // HLSL built-in

float spec = pow(max(0, dot(R, V)), _Shininess);  // _Shininess: 16-256 typical

float3 specular = _LightColor * _SpecularColor * spec;
float3 diffuse = _LightColor * _AlbedoColor * max(0, dot(N, L));

return float4(diffuse + specular, 1.0);
Shininess Value Appearance Material Example
8-16 Soft, wide highlight Rubber, skin
32-64 Medium highlight Plastic, painted metal
128-256 Sharp, tight highlight Polished metal, glass

Blinn-Phong: The Optimized Standard

Blinn-Phong is a tweak to Phong that's cheaper to compute and often looks better at grazing angles. It's the default in many engines.

The Half Vector

Instead of calculating reflection R, use the HALF VECTOR H:

    L       H       V
     \      ↑      /
      \     |     /
       \    |    /
        \   |   /
─────────●──────────  SURFACE
         ↑
         N

H = normalize(L + V)  // Halfway between light and view
specular = pow(max(0, dot(N, H)), shininess)

Why it's better:

  • No reflect() call needed
  • More stable at grazing angles
  • Widely used as industry standard

HLSL Implementation

float3 N = normalize(input.worldNormal);
float3 L = normalize(_LightPosition - input.worldPos);
float3 V = normalize(_CameraPosition - input.worldPos);

float3 H = normalize(L + V);  // Half vector

float NdotL = max(0, dot(N, L));
float NdotH = max(0, dot(N, H));

float3 diffuse = _LightColor * _AlbedoColor * NdotL;
float3 specular = _LightColor * _SpecularColor * pow(NdotH, _Shininess);

return float4(diffuse + specular, 1.0);

Ambient: The Fake Fill Light

Pure Lambert/Phong leaves shadowed areas completely black. In reality, light bounces around (global illumination). Ambient is a cheap approximation.

float3 ambient = _AmbientColor * _AlbedoColor;  // Constant term
float3 diffuse = _LightColor * _AlbedoColor * NdotL;
float3 specular = _LightColor * _SpecularColor * pow(NdotH, _Shininess);

float3 finalColor = ambient + diffuse + specular;

The classic lighting equation:

Final Color = Ambient + Diffuse + Specular
            = (Ia * Ka) + (Id * Kd * NdotL) + (Is * Ks * spec)
            
Where:
  Ia, Id, Is = ambient, diffuse, specular light intensity
  Ka, Kd, Ks = ambient, diffuse, specular material color

Fresnel: Edge Glow

Fresnel (pronounced "freh-NEL") describes how surfaces reflect more light at grazing angles. Look at a table from above—you see the wood. Look at it edge-on—you see reflections.

👁️ View from above:     👁️ View at angle:
     |                        \
     V                         \
─────────────              ─────────────
See surface color          See more reflection
(low fresnel)              (high fresnel)

Schlick's Approximation

float3 V = normalize(_CameraPosition - input.worldPos);
float3 N = normalize(input.worldNormal);

float fresnel = pow(1.0 - max(0, dot(N, V)), _FresnelPower);

// Use for rim lighting, environmental reflections, etc.
float3 rimLight = fresnel * _RimColor;

Use cases: Rim lighting on characters, environmental reflections, water surfaces, energy shields, sci-fi effects.

Normal Mapping: Per-Pixel Detail

Now we can explain where tangent comes in. Normal maps encode surface detail as colors—but those normals are in tangent space, not world space.

Tangent Space (TBN)

TANGENT SPACE for each vertex:
        
        N (normal - perpendicular to surface)
        ↑
        |
        |
        ●───→ T (tangent - along UV.x direction)
       /
      /
     ↙ B (bitangent - along UV.y direction)

TBN matrix transforms from tangent space to world space

Normal maps store normals pointing "up" as (0.5, 0.5, 1.0)—that lavender/blue color. The shader:

  1. Samples the normal map
  2. Decodes it from [0,1] to [-1,1] range
  3. Transforms from tangent space to world space using TBN matrix
  4. Uses the result for lighting calculations
// Simplified normal mapping
float3 tangentNormal = tex2D(_NormalMap, input.uv).xyz * 2.0 - 1.0;  // Decode
float3x3 TBN = float3x3(input.worldTangent, input.worldBitangent, input.worldNormal);
float3 worldNormal = normalize(mul(tangentNormal, TBN));  // Transform

// Now use worldNormal for lighting
float NdotL = max(0, dot(worldNormal, L));

Putting It Together: A Complete Shader

// Fragment shader with Blinn-Phong + Fresnel
float4 frag(Varyings input) : SV_Target
{
    // Sample textures
    float4 albedo = tex2D(_MainTex, input.uv) * _Color;
    
    // Vectors (all normalized, all pointing "outward")
    float3 N = normalize(input.worldNormal);
    float3 L = normalize(_LightPos - input.worldPos);
    float3 V = normalize(_CameraPos - input.worldPos);
    float3 H = normalize(L + V);
    
    // Lighting terms
    float NdotL = max(0, dot(N, L));
    float NdotH = max(0, dot(N, H));
    float NdotV = max(0, dot(N, V));
    
    // Diffuse (Lambert)
    float3 diffuse = _LightColor * albedo.rgb * NdotL;
    
    // Specular (Blinn-Phong)
    float3 specular = _LightColor * _SpecColor * pow(NdotH, _Shininess);
    
    // Fresnel rim
    float fresnel = pow(1.0 - NdotV, _FresnelPower);
    float3 rim = fresnel * _RimColor;
    
    // Ambient
    float3 ambient = _AmbientColor * albedo.rgb;
    
    // Combine
    float3 final = ambient + diffuse + specular + rim;
    return float4(final, albedo.a);
}

Beyond Blinn-Phong: PBR Preview

Modern engines use Physically Based Rendering (PBR), which replaces the ad-hoc Phong model with energy-conserving formulas. Key concepts:

  • Metallic/Roughness workflow: Two parameters control surface behavior instead of separate diffuse/specular colors
  • Energy conservation: Total reflected light ≤ incoming light
  • Microfacet theory: Models surface as tiny mirrors at various angles
  • Cook-Torrance BRDF: The math behind modern specular

Understanding Lambert, Phong, and Fresnel gives you the foundation to understand PBR—which builds on these same dot products with more sophisticated distribution functions.

Key Takeaways

  1. Interpolation is how pixel data comes from vertices—GPU automatically interpolates all vertex outputs using barycentric coordinates
  2. Lambert diffuse = N · L (clamped). Simulates light scattering on matte surfaces
  3. Half-Lambert = (N · L) * 0.5 + 0.5. Softens shadows for stylized look
  4. Phong specular uses reflection vector R. Sharp highlights on shiny surfaces
  5. Blinn-Phong uses half vector H = normalize(L + V). Industry standard, cheaper than Phong
  6. Fresnel = (1 - N · V)^power. Edge glow, rim lighting
  7. TBN matrix transforms normal map data from tangent space to world space—that's where tangent is used
  8. Ambient + Diffuse + Specular is the classic lighting equation
