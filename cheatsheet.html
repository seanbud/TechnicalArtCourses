All key concepts at a glance for your VRify interview

1. Unity Engine Loop

Execution Order

  • Awake/OnEnable/Start — Initialization (once)
  • FixedUpdate — Physics (0, 1, or many per frame)
  • Update — Game logic (once per frame)
  • LateUpdate — Post-update (cameras)
  • Rendering — Culling, draw calls, shaders

Update vs FixedUpdate

  • FixedUpdate: Fixed timestep (0.02s), for physics/Rigidbody
  • Update: Variable deltaTime, for input/non-physics
  • Use Time.deltaTime in Update for frame-independence
  • FixedUpdate runs multiple times if frame is slow

2. Render Pipeline

Flow: Frustum Culling → Occlusion Culling → Sort by Render Queue → Batch → GPU (Vertex → Rasterize → Fragment) → Composite

Culling

  • Frustum: Skip objects outside camera view (uses AABB)
  • Occlusion: Skip objects hidden behind others (needs baking)
  • Renderer.bounds defines culling volume

Batching ("Changing Brushes")

  • Each material change = draw call
  • Static Batching: Combine static meshes at build
  • GPU Instancing: Many copies, one call
  • SRP Batcher: Reduces per-call CPU cost (not count)

3. Shaders

Pipeline: Vertex Shader (per-vertex) → [Geometry*] → Rasterization (HW) → Fragment Shader (per-pixel) → Depth Test → Blend

Key Concepts

  • Shader = multiple programs (vertex, fragment, etc.)
  • Vertex shader: Transform positions, pass data to frag
  • Fragment shader: Calculate pixel color — main bottleneck
  • Pass: One complete render; multi-pass = multiple renders

Compilation

  • ShaderLab wraps HLSL in Unity format
  • Compiled at build time for target platform
  • Variants: Different combos of keywords — can explode build size
  • Use shader_feature over multi_compile to strip unused

GBuffer: Stores per-pixel surface data (albedo, normals, depth, roughness) so lights process only visible pixels.
Forward+: Clustered light culling — divides screen into tiles, each tile only processes relevant lights.

5. URP Settings Quick Ref

Recommended for Browser

  • Render Path: Forward
  • HDR: Off
  • MSAA: 2x or Off (use FXAA)
  • Shadow Res: 512-1024
  • Shadow Cascades: 1-2
  • SRP Batcher: Enabled

MSAA Options

  • Off: Use FXAA post-process instead
  • 2x: Good balance for browser
  • 4x/8x: Too expensive for WebGL
  • Note: MSAA doesn't work with some screen-space effects

6. GPU Architecture

SIMD & Warp Divergence

  • GPUs execute same instruction on many threads (warp/wavefront)
  • Branching hurts: If threads diverge, both branches execute
  • Uniform branches (same value for all) are OK
  • Keep fragment shaders simple — runs millions of times

Texture2D Arrays

  • Stack of same-size textures, sample with (u, v, layer)
  • Enables batching with texture variety
  • Use for: terrain blending, character skins, decals
Memory Hierarchy (fast→slow): Registers → Shared Memory → L1/L2 Cache → Texture Cache → Global VRAM
Compute Shaders: General GPU compute, not for rendering directly. WebGL 2.0 does NOT support them.

7. Mesh Fundamentals

Components

  • MeshFilter: Holds mesh data (geometry)
  • MeshRenderer: Draws mesh with materials
  • .sharedMesh = shared asset (read-only)
  • .mesh = creates instance copy (for modification)

Vertex Attributes

  • Position: 3D location
  • Normal: Surface direction (lighting)
  • UV: Texture coordinates
  • Tangent: For normal mapping
  • SubMesh: Portion with different material

8. UI Rendering

Canvas Modes

  • Screen Space Overlay: Always on top, ignores cameras
  • Screen Space Camera: Rendered by camera, can have depth
  • World Space: 3D positioned, affected by perspective

Optimization

  • Canvas rebuilds when any element changes — expensive!
  • Separate canvases: Static vs dynamic elements
  • Overdraw: Stacked UI draws pixels multiple times
  • Disable Raycast Target on decorative elements
  • Use TextMeshPro for text (SDF, crisp at any size)
Batching breaks: Different textures, different materials, interleaved hierarchy order. Group by material in hierarchy!

10. Mesh Optimization

Algorithms

  • Decimation (QEM): Collapse edges with lowest error
  • Hole Detection: Find boundary edges (only 1 adjacent tri)
  • LOD: Multiple resolutions, switch by distance
  • Vertex Welding: Merge vertices within tolerance

Best Practices

  • Preserve sharp edges and UV seams during decimation
  • LOD levels: 100%, 50%, 25%, 10% of triangles
  • Triplanar mapping avoids need for UVs on CAD
  • Validate meshes: watertight, consistent normals

11. WebGL Constraints

Key Limits: Single-threaded | ~2GB memory | No compute shaders | Download before run | Shader stalls

Memory

  • Textures dominate — 4K texture = 64MB!
  • Use ETC2 compression (WebGL 2.0 guaranteed)
  • Max 1K-2K textures for most assets
  • Unload unused assets aggressively

Load Time

  • Enable Brotli/gzip compression
  • Use Addressables for streaming
  • Strip unused engine code
  • Shader warmup during loading screen

12. Unity Asset Transform (UAT)

What It Does

  • Converts CAD formats (CATIA, NX, STEP, IGES) to Unity
  • Tessellates NURBS to triangles with quality control
  • Decimates, generates LODs, maps materials
  • Preserves hierarchy and metadata

Industrial Scale

  • Scriptable API for automated pipelines
  • "Excited to learn VRify's specific implementation"

13. Editor Tooling & Automation

Types of Tools

  • Menu Items: Custom buttons in the top bar
  • Inspectors: Customize how components look
  • Editor Windows: Full custom floating panels
  • Postprocessors: Hooks that run on asset import

AssetPostprocessor

  • Hooks: OnPostprocessModel, OnPostprocessTexture
  • Purpose: Fix coordinate shifts, map materials, automate UAT
  • Standardize "garbage" client data upon import

The API Rules

  • Editor Folder: Scripts using UnityEditor must be in Editor/
  • AssetDatabase: Programmatic access to all files
  • ModelImporter: Control decimation/UVs via code

VRify-Specific Quick Reference

Browser Build Checklist

  • ✅ Forward rendering (not Deferred)
  • ✅ MSAA 2x or FXAA post-process
  • ✅ Shadow resolution 512-1024, 1-2 cascades
  • ✅ ETC2 texture compression
  • ✅ Max 1K-2K textures
  • ✅ SRP Batcher enabled
  • ✅ Brotli/gzip compression on server
  • ✅ Shader warmup on load
  • ✅ Origin-shift large geospatial coordinates
  • ✅ Object pooling (avoid GC)
