FishNet Cheat Sheet

Quick reference for architecture, attributes, and CSP logic.

Core Attributes

  • [ServerRpc(RequireOwnership = false)]: Invoked by Client, executed on Server. RequireOwnership prevents hacking but limits who can call it (e.g. interacting with world objects).
  • [ObserversRpc(BufferLast = true)]: Invoked by Server, executed on Clients. BufferLast ensures late-joining players receive the last fired RPC.
  • [TargetRpc]: Invoked by Server, executed on ONE specific Client (requires passing a NetworkConnection).

SyncTypes (State)

Used to sync variables from Server to Clients automatically.

  • SyncVar<T>: Syncs primitive values (int, float, bool, Vector3). Exposes OnChange event.
  • SyncList<T>: Syncs collections. Includes events for Add, Remove, Clear.
  • SyncObject: Custom complex structs synced entirely.

Client-Side Prediction (CSP)

  • [Replicate]: Executes moving logic. Runs on Predictor + Server.
  • [Reconcile]: Returns ground-truth state from Server to Client. Snaps the client to the server's perspective.
  • TimeManager.OnTick: Bind here to construct Replicate data structurally (avoids Update() float jitter).

NetworkBehaviour Properties

  • IsServerInitialized: True if the code is currently running on the server.
  • IsClientInitialized: True if running on the client.
  • IsOwner: True if the local player has authority over this specific game object.
  • OwnerId: The integer ID of the connection that owns the object.

Lifetime Hooks

  • public override void OnStartNetwork() (Wait for network initialize)
  • public override void OnStartServer() (Server logic init)
  • public override void OnStartClient() (Client logic init)
  • public override void OnOwnershipClient(NetworkConnection prevOwner) (Authority changed)
