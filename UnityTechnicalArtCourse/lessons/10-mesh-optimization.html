Mesh Optimization Algorithms

Decimation, hole detection, LOD, and mesh repair

Why Mesh Optimization Matters

CAD data and scanned meshes are rarely optimized for real-time rendering. A mechanical part might have millions of triangles when thousands would visually suffice. For browser targets, aggressive optimization is essential.

🧠 The Triangle Budget

Desktop games: ~1-5 million triangles per frame
Mobile games: ~100K-500K triangles per frame
Browser (WebGL): ~100K-300K triangles per frame (conservative)

A single CAD assembly can exceed these budgets before any scene composition!

Mesh Decimation

Decimation reduces polygon count while preserving shape. The most common algorithm is Quadric Error Metrics (QEM).

How QEM Works (Conceptual)

QEM DECIMATION PROCESS: Original: After Decimation: /\ /\ / \ / \ /____\ / \ |\ /| | | | \/ | ───────────▶ | | | /\ | | | |/__\| |____| Algorithm: 1. For each edge, calculate "error" of collapsing it (Error = how much shape changes) 2. Sort edges by error (lowest first) 3. Collapse lowest-error edge 4. Update neighboring edge errors 5. Repeat until target polygon count reached

Quadric Error Explained

Each vertex has associated planes (from its neighboring triangles). A "quadric" matrix represents how far a point deviates from these planes. When collapsing two vertices:

  • Sum their quadric matrices
  • Find optimal position for merged vertex (minimizes error)
  • The error value determines priority
💡 Preservation Weights

Good decimation algorithms support feature preservation:
• Sharp edges (detect angle threshold)
• UV seams (prevent texture distortion)
• Material boundaries (keep submesh integrity)
Without these, simplified meshes look wrong.

Decimation in Unity

  • Unity Asset Transform: Industrial solution with quality decimation
  • Simplygon: Industry standard (paid license)
  • MeshSimplifier: Open-source C# package (quality varies)
  • Blender API: Server-side processing via Blender's Decimate modifier

Hole Detection and Filling

CAD exports and scans often have holes — missing triangles in the mesh surface. These cause rendering artifacts and break operations like boolean CSG.

What Is a Hole?

MESH HOLES: "Watertight" mesh: Mesh with hole: ┌───────────┐ ┌───────────┐ │ │ │ │ │ (solid) │ │ (gap) │ │ │ │ ○ │ └───────────┘ └───────────┘ All edges shared Some edges have by exactly 2 triangles only 1 triangle = "manifold" = "boundary edges"

Hole Detection Algorithm

  1. Build edge adjacency map (which triangles share each edge)
  2. Find boundary edges — edges with only one adjacent triangle
  3. Link boundary edges into loops (each loop is a hole)
  4. Small loops might be holes; large loops might be intentional openings

Hole Filling Approaches

  • Simple: Triangulate the boundary loop (fan or ear-clipping)
  • Smooth: Fill then apply smoothing to blend with surroundings
  • Curvature-aware: Match surrounding surface curvature
💚 VRify Relevance

Mining equipment CAD often has holes where components interface. Decide whether to fill holes (makes mesh "solid") or leave them (preserves original data). Document the choice in your pipeline.

Level of Detail (LOD)

LOD uses multiple mesh resolutions, switching based on distance. Essential for large scenes.

LOD CONCEPT: Distance: Near Mid Far ◀──────────────────────────────────▶ LOD Level: LOD0 LOD1 LOD2 Triangles: 10,000 2,500 500 Detail: Full 75% reduced 95% reduced Unity's LOD Group component handles switching

LOD Generation

  • Apply decimation at increasing ratios (25%, 50%, 75%, 90%)
  • Set distance thresholds based on object size and screen percentage
  • Test transitions for visual popping — adjust thresholds if jarring
  • Consider cross-fade transitions (URP supports this)

LOD Bias and Screen Size

Unity uses screen percentage to determine LOD:

  • LOD 0 when object covers > 60% of screen height
  • LOD 1 when covering 30-60%
  • LOD 2 when covering 15-30%
  • Culled when below threshold

Mesh Repair Operations

Common Problems and Fixes

Problem Detection Solution
Degenerate triangles Zero area, collinear vertices Remove or merge vertices
Duplicate vertices Multiple vertices at same position Weld with tolerance
Flipped normals Inconsistent winding Recalculate or flip per-face
Non-manifold geometry Edges shared by >2 triangles Duplicate/separate geometry
Self-intersections Triangles passing through each other Boolean union or manual fix

Vertex Welding

CAD exports often have "open" vertices that should be shared. Welding merges vertices within a distance threshold:

// Conceptual welding
foreach (vertex in mesh) {
    neighbors = FindVerticesWithinDistance(vertex, tolerance);
    if (neighbors.Count > 0) {
        MergeToSingleVertex(vertex, neighbors);
    }
}

UV Optimization

For CAD data often lacking UVs:

  • Automatic UV Generation: Box projection, cylindrical, spherical mapping
  • UV Unwrapping: More complex, creates proper charts
  • Triplanar Mapping: Skip UVs entirely, sample textures based on world position
🧠 Triplanar Advantage

Triplanar shaders project textures from 3 axes and blend based on surface normal. No UV required. Perfect for terrain, rocks, and geometry without proper UVs. Small performance cost but huge workflow benefit.

Algorithm Complexity Overview

Algorithm Complexity Notes
QEM Decimation O(n log n) Priority queue for edge collapses
Hole Detection O(e) edges Linear scan of edges
Hole Filling (simple) O(b) boundary Triangulate boundary loop
Vertex Welding O(n log n) Spatial data structure (octree)
Normal Recalculation O(n) Iterate triangles once

Key Takeaways

  • Decimation reduces triangles; QEM preserves shape by minimizing geometric error
  • Feature preservation (edges, UVs, materials) prevents visual artifacts
  • Hole detection finds boundary edges; hole filling triangulates the gap
  • LOD uses distance-based mesh switching for performance
  • Mesh repair: welding, normal fixing, degenerate removal
  • Triplanar mapping is a practical solution for UV-less CAD data
  • Unity Asset Transform handles many of these operations at industrial scale
