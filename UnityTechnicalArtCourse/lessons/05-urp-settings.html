URP Deep Dive

Render features, quality settings, and browser optimization

What Is URP?

The Universal Render Pipeline is Unity's default scriptable render pipeline, designed for cross-platform performance. It replaced the old Built-in render pipeline for most use cases.

UNITY RENDER PIPELINES: Built-in (Legacy) URP HDRP ───────────────── ─── ──── Fixed pipeline Scriptable, High fidelity, Not extensible extensible, compute-heavy, Still works cross-platform PC/Console only ↑ What VRify should use

Why URP for VRify

  • WebGL support: HDRP doesn't support WebGL; URP does
  • Scalability: Can target low-end devices while still looking good
  • SRP Batcher: Efficient draw call handling
  • Customizable: Render Features let you add custom passes

URP Asset Settings

The Universal Render Pipeline Asset controls global rendering behavior. Key settings to understand:

Rendering Section

Setting Options Recommendation for Browser
Render Path Forward, Forward+, Deferred Forward — safest for WebGL
Depth Texture Off/On On only if needed (some effects require it)
Opaque Texture Off/On Off unless using refraction effects

Quality Section

Setting What It Does Browser Recommendation
HDR High dynamic range rendering Off — saves memory, WebGL support varies
Anti Aliasing (MSAA) Multi-sample anti-aliasing 2x or Off — see MSAA section below
Render Scale Internal resolution multiplier 0.75-1.0 based on performance needs

MSAA: Understanding Anti-Aliasing

MSAA (Multi-Sample Anti-Aliasing) smooths jagged edges by rendering at multiple sample points per pixel.

MSAA Options in URP

Level Samples Memory Cost Use Case
Disabled 1 Base Mobile/low-end, or using post-process AA
2x 2 2x framebuffer Good balance for browser
4x 4 4x framebuffer High quality, desktop
8x 8 8x framebuffer Maximum quality, heavy
⚠️ MSAA and Post-Processing

MSAA doesn't work well with screen-space effects that read the depth buffer. If you're using SSAO, Depth of Field, or similar effects, consider FXAA or SMAA as post-process alternatives instead of MSAA.

Alternative AA Methods

  • FXAA: Fast, cheap, slightly blurry. Good for low-end.
  • SMAA: Better edge detection than FXAA, more expensive.
  • TAA: Temporal AA, uses motion vectors. Can cause ghosting but handles thin geometry well.
💚 VRify Recommendation

For browser: Try 2x MSAA first. If memory is tight or you're using depth-based effects, switch to FXAA as a post-process effect. Test on target browsers to verify performance.

Shadows in URP

Shadow settings heavily impact performance:

Setting Impact Browser Setting
Shadow Resolution Shadow map texture size 512-1024 (lower is faster)
Shadow Distance How far shadows render Keep minimal for scene needs
Shadow Cascades Quality vs distance balance 1-2 cascades max
Soft Shadows Blurred shadow edges Off or Low quality
💡 Shadow Cascades Explained

Cascades split the shadow map into near/far regions. Near shadows get more resolution. 4 cascades looks best but costs 4x the shadow map work. For browser, 1-2 cascades is usually sufficient for visualization purposes.

The SRP Batcher

One of URP's biggest performance wins. Understanding it is important:

WITHOUT SRP BATCHER: WITH SRP BATCHER: For each draw call: Once at setup: ├── Upload material properties └── Upload all material data ├── Set shader to persistent GPU buffer ├── Set textures ├── Draw For each draw call: ├── Just update transform Upload happens EVERY DRAW ├── Draw = Lots of CPU → GPU transfer Minimal per-draw overhead

Key insight: SRP Batcher doesn't reduce draw call count — it reduces the cost per draw call by keeping material data on the GPU.

Requirements for SRP Batcher

  • Shader must be SRP Batcher compatible (most URP shaders are)
  • Check in Shader Inspector: "SRP Batcher: compatible"
  • Enable in URP Asset settings
  • Different materials of the same shader can be batched

Render Features

URP's extensibility mechanism. A Render Feature is custom code that injects into the render pipeline at specific points.

Common Uses

  • Custom post-processing effects
  • Outline/silhouette rendering
  • Additional passes (depth prepass, etc.)
  • Render texture setup for custom effects

Built-in Render Features

  • Screen Space Ambient Occlusion (SSAO) — adds depth/shadow in corners
  • Decals — project textures onto surfaces
  • Render Objects — render specific layers with different settings

Camera Stacking

URP supports multiple cameras rendering in layers:

CAMERA STACKING: ┌─────────────────────────────────────┐ │ Overlay Camera (UI) │ ← Renders on top, no clear ├─────────────────────────────────────┤ │ Base Camera (3D Scene) │ ← Clears and renders world └─────────────────────────────────────┘ ↓ Final composite output
  • Base Camera: First camera, clears the screen
  • Overlay Camera: Renders on top, doesn't clear
  • Set Overlay cameras in Base camera's "Stack" list
  • Useful for: UI, picture-in-picture, split effects

Browser-Specific URP Settings

For WebGL deployment, optimize aggressively:

Recommended Browser Configuration

  • Render Path: Forward
  • HDR: Off
  • MSAA: 2x or Off (use FXAA post-process)
  • Render Scale: 0.75-1.0
  • Shadow Resolution: 512-1024
  • Shadow Distance: Minimal for needs
  • Shadow Cascades: 1-2
  • Soft Shadows: Off or Low
  • Additional Lights: Per Vertex or disabled
  • Depth/Opaque Textures: Only if needed
  • SRP Batcher: Enabled

Key Takeaways

  • URP is the right choice for cross-platform including WebGL
  • MSAA provides hardware AA but costs memory; 2x is a good balance
  • SRP Batcher reduces per-draw-call cost, not draw call count
  • Shadows are expensive; minimize resolution, distance, and cascades for browser
  • Render Features allow custom pipeline injection for effects
  • Camera Stacking enables layered rendering (useful for UI)
  • For browser: keep settings conservative, test on real browsers
