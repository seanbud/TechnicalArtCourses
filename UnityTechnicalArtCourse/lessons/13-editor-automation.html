Lesson 13

Unity Editor Scripting & Automation

Building tools to manage scale and technical standards

Overview: Extension over Modification

As a Technical Artist, your job isn't just to make art look good—it's to make the process of making art efficient. Unity is designed to be "hacked." Instead of modifying the engine source, we use the UnityEditor namespace to extend the interface.

TYPES OF EDITOR EXTENSIONS: 1. Menu Items - Trigger code from the top bar (e.g. "VRify/Cleanup Scene") 2. Custom Inspectors - Change how a Component looks in the Inspector 3. Editor Windows - Full custom floating panels (Dashboards) 4. Postprocessors - "Invisible" hooks that run when files are imported 5. Property Drawers - Change how specific data types (like [Range]) appear
💡 The "Editor" Folder Rule

Code that uses UnityEditor cannot be included in a final game build (it will fail to compile). You must place these scripts inside a folder named Editor. Unity's build system automatically strips these folders when you hit "Build for WebGL."

1. The GUI Core: EditorWindow

An EditorWindow is a window you can dock anywhere in the Unity UI. This is where you build "Control Centers."

using UnityEditor;
using UnityEngine;

// This class creates a new window in the Unity menu
public class VRifyManagerWindow : EditorWindow
{
    [MenuItem("VRify/Project Dashboard")]
    public static void ShowWindow() => GetWindow("VRify Dashboard");

    void OnGUI()
    {
        // All UI logic goes here (Buttons, Sliders, Labels)
        GUILayout.Label("Geospatial Ingestion Tool", EditorStyles.boldLabel);
        
        if (GUILayout.Button("Normalize Selected Coordinates"))
        {
            // Call your logic here!
        }
    }
}

2. Silent Automation: AssetPostprocessor

This is arguably the most important tool for VRify. When handling 250+ clients, you cannot manually check every file. AssetPostprocessor lets you intercept assets as they enter the project.

POSTPROCESSOR FLOW: Client .FBX Arrives → OnPreprocessModel → (Change Import Settings) ↓ Unity Imports Data ↓ OnPostprocessModel → (Modify GameObject/Hierarchy) ↓ Asset Ready in Project

Example: The Automated Coordinate Fix

public class AutoIngest : AssetPostprocessor
{
    // Catch the model before it's even fully "in" the project
    void OnPreprocessModel()
    {
        ModelImporter importer = (ModelImporter)assetImporter;
        
        // Auto-set settings for WebGL
        importer.meshCompression = ModelImporterMeshCompression.Medium;
        importer.optimizeMesh = true;
    }

    // Catch the model after Unity turns it into a GameObject
    void OnPostprocessModel(GameObject g)
    {
        // Automatically apply the coordinate origin shift (Lesson 09)
        // so the mesh is centered at (0,0,0) instead of 5,000km away.
        ShiftToGeospatialOrigin(g);
    }
}

3. Useful API "Heavy Hitters"

Key classes you'll use to build these tools:

Class Purpose
AssetDatabase The master index. Use it to find, load, move, or rename assets via code.
ModelImporter The "knobs and dials" for 3D files. Controls decimation, UV generation, and scale.
Selection Lets your tool know what the user has currently clicked on in the project.
EditorUtility Helpers for displaying progress bars, save file dialogs, and dirty flags.
Handles Drawing custom 3D visuals (lines, icons) directly in the Scene View.

4. Application at Scale (250 Clients)

In your interview, don't just say you can "write code." Say you can Standardize the Pipeline:

  • Validation: Write a tool that scans a client's folder and flags any models that are over the "WebGL Triangle Budget."
  • Remapping: Create a script that looks at CAD material names (e.g. "COLOR_251") and automatically swaps them for your high-quality WebGL shaders.
  • Batch Processing: An EditorWindow that lets a producer select 50 folders and click "Optimize All" using the Unity Asset Transform API.
💚 VRify Interview Angle

"I prioritize building tools that fail early. By using AssetPostprocessor and custom validation windows, we can catch ingestion errors immediately rather than at runtime in the browser where they are harder to debug."

Key Takeaways

  • UnityEditor namespace code lives in Editor/ folders.
  • EditorWindow provides the UI for your custom dashboards.
  • AssetPostprocessor automates the cleaning of "garbage" incoming data.
  • AssetDatabase is how you manipulate files programmatically.
  • Automation is the only way to handle 250+ uncoordinated clients effectively.
