Data Strategy & SQL

Managing Petabytes: Tiered Storage, Robust Validation, and Asset Databases.

The Digital Landfill

A mocap stage produces 4TB of raw video per day. If you calculate that out: 4TB * 200 Days = 800TB per year. You cannot store 800TB on fast NVMe drives. It would cost millions.

You need a Lifecycle Strategy. We move data based on "Temperature".

1. Implementation: Robust Validation (The Validator Class)

Moving data is dangerous. A simple copy script is not enough. You need Retries and Checksums.

Python: The Validator

import hashlib
import shutil
import os
import time
import logging

class AssetValidator:
    def __init__(self):
        logging.basicConfig(level=logging.INFO)
        self.logger = logging.getLogger("DataMover")

    def calculate_md5(self, filepath):
        """Streams file to avoid loading 2GB into RAM"""
        hash_md5 = hashlib.md5()
        with open(filepath, "rb") as f:
            for chunk in iter(lambda: f.read(4096), b""):
                hash_md5.update(chunk)
        return hash_md5.hexdigest()

    def safe_move(self, source, dest, retries=3):
        for attempt in range(retries):
            try:
                # 1. Calc Source Hash
                src_hash = self.calculate_md5(source)
                
                # 2. Copy
                shutil.copy2(source, dest)
                
                # 3. Calc Dest Hash
                dst_hash = self.calculate_md5(dest)
                
                # 4. Compare
                if src_hash == dst_hash:
                    self.logger.info(f"Success: {source} -> {dest}")
                    return True
                else:
                    self.logger.error(f"Corruption Detected! Attempt {attempt+1}")
                    os.remove(dest) # Clean up bad file
                    
            except Exception as e:
                self.logger.error(f"IO Error: {e}")
                time.sleep(1) # Backoff before retry
                
        raise Exception("Failed to move file after 3 attempts.")

2. Architecture: The Asset Database (SQL)

Filesystems are slow to search. If you have 1 million files, `os.walk()` takes 10 minutes. You need a Database to query where things are instantly.

Python: Tracking Assets with SQLite

import sqlite3

def init_db():
    conn = sqlite3.connect('assets.db')
    c = conn.cursor()
    # Create Table
    c.execute('''CREATE TABLE IF NOT EXISTS shots (
                    id INTEGER PRIMARY KEY,
                    shot_name TEXT,
                    path TEXT,
                    tier TEXT,
                    last_accessed DATE
                )''')
    conn.commit()
    return conn

def register_asset(shot_name, path):
    conn = init_db()
    c = conn.cursor()
    c.execute("INSERT INTO shots (shot_name, path, tier, last_accessed) VALUES (?, ?, ?, ?)",
              (shot_name, path, 'HOT', '2023-10-27'))
    conn.commit()
    print(f"Registered {shot_name} in Database.")

def find_cold_assets():
    """Find assets that haven't been touched in 30 days"""
    conn = init_db()
    c = conn.cursor()
    c.execute("SELECT path FROM shots WHERE last_accessed < date('now','-30 days') AND tier='HOT'")
    return c.fetchall()

4. Technical Drill: Interview Prep

🗣️ Interview Prompt

"Design a system to delete old files without deleting something important."

The Answer: The "Sweeper" Service.

Never rely on file dates alone (opening a file updates the date). Use the Database.

  1. Query: Select files marked "Approved" (Final) that are older than 6 months.
  2. Verify: Check if they exist on the "Cold Storage" (Tape/Cloud) first.
  3. Delete: Remove from "Hot Storage" only if the backup is confirmed.
  4. Update DB: Change status from 'HOT' to 'COLD'.

Key Takeaways

  • Tiered Storage: Move data from Hot (SSD) to Warm (HDD) to Cold (Tape) based on age to save money.
  • Hashing (MD5): The only way to mathematically prove a file transferred correctly. Bit rot is real.
  • Asset Database: Use SQLite or similar to track files. `os.walk` is too slow for 100TB.
  • Filename Schema: Enforce strict naming (Project_Date_ShotID) so files are sortable and queryable.
  • The Sweeper: Automate data lifecycle. Don't rely on humans to clean up.
🎬 EA Studio Relevance: Dead Space

In Dead Space, audio files are massive uncompressed WAVs. We can't keep them all on the fast SSDs (Tier 1 Storage). We use a Tiered Storage Policy. After 30 days of inactivity, a script moves the raw audio to the slow NAS (Tier 2). After 90 days, it goes to Amazon Glacier (Cold Storage). This saves the studio $50,000/year in storage costs.
