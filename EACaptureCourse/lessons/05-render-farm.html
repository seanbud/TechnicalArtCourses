Buying Time with Money

A single capture shot needs 10 minutes to process (Solve + Retarget + Render). We shoot 600 shots a day. That's 100 hours of compute time. You can't do that on one workstation.

We use AWS Thinkbox Deadline (or OpenCue) to manage a farm of 100+ nodes.

1. Implementation: Submitting a Job (Python)

In a pipeline, you don't use the Monitor UI. You write scripts that submit jobs automatically. A "Job" in Deadline is just two text files: JobInfo and PluginInfo.

Python: The Submission Script

import subprocess
import os

def submit_maya_job(scene_file, start, end, output_path):
    # 1. Define Job Info (Metadata)
    job_info = [
        "Plugin=MayaBatch",
        "Name=Render_Shot_01",
        "Comment=Nightly Build",
        "Pool=gpu_nodes",
        "Frames={}-{}".format(start, end),
        "ChunkSize=10" # Optimization!
    ]
    
    # 2. Define Plugin Info (Arguments for Maya)
    plugin_info = [
        "SceneFile={}".format(scene_file),
        "OutputFilePath={}".format(output_path),
        "Version=2024",
        "Renderer=Arnold"
    ]
    
    # 3. Write files to temp
    with open("job_info.txt", "w") as f:
        f.write("\n".join(job_info))
        
    with open("plugin_info.txt", "w") as f:
        f.write("\n".join(plugin_info))
        
    # 4. Call Deadline Command
    # deadlinecommand.exe job_info.txt plugin_info.txt
    result = subprocess.check_output(
        ["deadlinecommand", "job_info.txt", "plugin_info.txt"]
    )
    print("Job Submitted! ID: " + result.decode())
    return result.decode().strip()

2. Architecture: The Directed Acyclic Graph (DAG)

A simple list of jobs isn't enough. Job B can't start until Job A finishes. This is a Dependency Graph.

3. Idempotency: The Re-Run Rule

Computers crash. Networks fail. A job will fail eventually. The farm software will retry it automatically.

рҹ§  Concept: Idempotency

Definition: Running a function multiple times produces the same result as running it once.
Bad: Appending "Success" to a log file. (Run twice = "SuccessSuccess")
Good: Overwriting the log file. (Run twice = "Success")
Why: If a node dies halfway through, the Retry must not corrupt the data.

4. Technical Drill: The Cloud Burst (Boto3)

Sometimes local render nodes aren't enough. You need to upload to AWS S3. You use Boto3.

Python: The "Uploader"

import boto3
from botocore.exceptions import NoCredentialsError

def upload_to_s3(local_file, bucket, s3_file):
    s3 = boto3.client('s3')
    
    try:
        # Upload with automatic multipart (fast for big files)
        s3.upload_file(local_file, bucket, s3_file)
        print("Upload Successful")
        return True
    except FileNotFoundError:
        print("The file was not found")
        return False
    except NoCredentialsError:
        print("Credentials not available")
        return False

Key Takeaways

  • Horizontal Scaling: It's cheaper to buy 100 slow computers than 1 supercomputer.
  • DAG (Directed Acyclic Graph): Jobs have dependencies. Render cannot start until Solve finishes.
  • Idempotency: A job must produce the same result even if run multiple times. Safeguard against crashes.
  • Chunking: Grouping frames together (1-10) to amortize the expensive startup time of Maya.
  • Job Submission: Use Python APIs (Deadline/OpenCue) to submit jobs, not the GUI.
