From Infrared Light to Rigid Body Solving: The Math Behind the Magic.

1. The Big Picture: Signal vs Noise

Motion Capture is data acquisition from the real world. Unlike hand-key animation (which is additive art), mocap is subtractive data processing. Your job is to filter the "Signal" (Actor) from the "Noise" (Reflections, Occlusion, Marker Swaps).

2. Implementation: The Vector3 Class

In a technical interview, you cannot rely on `maya.cmds` or `UnityEngine`. You must demonstrate that you understand the linear algebra under the hood. You should be comfortable writing a helper class like this from scratch.

Python: The Core Math Library

import math

class Vector3:
    def __init__(self, x, y, z):
        self.x = x
        self.y = y
        self.z = z

    def __sub__(self, other):
        return Vector3(self.x - other.x, self.y - other.y, self.z - other.z)

    def magnitude(self):
        return math.sqrt(self.x**2 + self.y**2 + self.z**2)

    def normalize(self):
        mag = self.magnitude()
        if mag == 0: return Vector3(0,0,0)
        return Vector3(self.x/mag, self.y/mag, self.z/mag)

    def dot(self, other):
        """Returns scalar: how much two vectors align."""
        return (self.x * other.x) + (self.y * other.y) + (self.z * other.z)

    def cross(self, other):
        """Returns vector: perpendicular to both inputs."""
        return Vector3(
            self.y*other.z - self.z*other.y,
            self.z*other.x - self.x*other.z,
            self.x*other.y - self.y*other.x
        )

# Example Usage
p1 = Vector3(0, 0, 0)
p2 = Vector3(10, 0, 0)
dist = (p2 - p1).magnitude() # Returns 10.0

3. Algorithm: Detecting Marker Swaps

A "Marker Swap" occurs when two markers pass close to each other (e.g., crossing arms) and the system mislabels them. The Left Wrist becomes the Right Wrist. This creates a physical impossibility: a wrist cannot move 5 meters in 0.01 seconds.

We detect this by analyzing the Velocity Spike.

Python: The Swap Detector

def detect_swaps(frames, threshold_speed=500.0):
    """
    frames: List of Vector3 positions for a SINGLE marker over time.
    threshold_speed: Max allowed speed (mm per frame).
    """
    spikes = []
    
    for i in range(1, len(frames)):
        current_pos = frames[i]
        prev_pos = frames[i-1]
        
        # Calculate distance traveled in one frame
        delta = current_pos - prev_pos
        speed = delta.magnitude()
        
        # If speed exceeds human possibility, it's a swap/error
        if speed > threshold_speed:
            spikes.append(i)
            print(f"CRITICAL: Frame {i} velocity {speed:.2f} exceeds limit!")
            
    return spikes

4. Deep Dive: Rigid Body Solving (SVD)

How do we turn floating dots into a bone? We fit a "Rigid Body" to them. This is typically done using Singular Value Decomposition (SVD).

🧠 Concept: The Kabsch Algorithm

1. Centroid: Find the center of the cloud of points.
2. Center: Move both the Source Cloud and Target Cloud so their centroids are at (0,0,0).
3. Rotate: Find the optimal Rotation Matrix (R) that aligns the points. This is where SVD is used.

Conceptual Implementation (Numpy Style)

import numpy as np

def solve_rigid_body(source_points, target_points):
    # 1. Calculate Centroids
    centroid_A = np.mean(source_points, axis=0)
    centroid_B = np.mean(target_points, axis=0)
    
    # 2. Center the points
    AA = source_points - centroid_A
    BB = target_points - centroid_B
    
    # 3. Covariance Matrix
    H = np.dot(AA.T, BB)
    
    # 4. SVD (The Magic)
    U, S, Vt = np.linalg.svd(H)
    
    # 5. Calculate Rotation
    Rotation = np.dot(Vt.T, U.T)
    
    return Rotation, centroid_B
🎬 EA Studio Relevance: Sports & Realism

Why does this math matter? In FIFA/FC, players are judged by their signature moves. If the rigid body solve is off by even 2mm, the foot slides on the grass ("foot skating"). Millions of players notice this immediately. High-fidelity solving is the barrier between a game that feels "arcade" and one that feels "simulation".

5. Technical Drill: Interview Prep

🗣️ Interview Prompt

"I have a character that needs to look at a target, but their head shouldn't rotate past 45 degrees. Write the logic."

The Solution:

  1. Construct a LookAt Vector (Target Pos - Head Pos).
  2. Calculate the Angle between the Character Forward Vector and the LookAt Vector using acos(dot_product).
  3. If Angle > 45, Clamp the rotation or use Spherical Linear Interpolation (Slerp) to limit it.

Key Takeaways

  • Motion Capture is Subtractive: You are filtering signal (performance) from noise (marker swaps, occlusion).
  • Dot Product (Facing Ratio): The mathematical tool for checking if a camera can see a marker.
  • Rigid Body Solving (SVD): We use Singular Value Decomposition to minimize error between the point cloud and the bone.
  • Vector3 Class: The fundamental data structure for all 3D math in python.
  • The 1mm Rule: If the residual error is > 1mm, the solve is bad. Calibration is king.
