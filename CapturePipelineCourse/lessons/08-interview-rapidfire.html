1. Infrastructure as Code (IaC)

The philosophy: Your pipeline IS your product. Treat it like software, not a collection of scripts on someone's desktop.

2. Data Gravity & The Capture-to-Engine Bridge

This is the single most important topic for this interview. "Data Gravity" = large data is hard to move. The bigger the file, the more it resists movement.

┌─────────────────────────────────────────────────────────────────┐ │ THE CAPTURE-TO-ENGINE HANDOFF │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ MONDAY (Stage) MONDAY NIGHT (Farm) TUESDAY (Artist) │ │ │ │ ┌──────────┐ ┌──────────────┐ ┌──────────────┐ │ │ │ RAW DATA │ ──→ │ AUTO-PROCESS │ ──→ │ GAME-READY │ │ │ │ 500GB │ Copy │ Solve+Export │ Push │ .fbx/.anim │ │ │ └──────────┘ to NAS └──────────────┘ to P4 └──────────────┘ │ │ │ │ │ │ │ File Watcher Deadline Farm P4 Auto-Submit │ │ (inotify/watchdog) (100 nodes) + Slack Notify │ │ │ │ Key: NO HUMAN TOUCH between Monday PM and Tuesday AM. │ │ The pipeline must be fully automated and idempotent. │ │ │ └─────────────────────────────────────────────────────────────────┘

The answer pattern:

  1. Ingest: Capture finishes → Sweeper bot (Python watchdog library) detects new files on the NAS.
  2. Validate: MD5 hash check → file size sanity → marker count check.
  3. Process: Submit to Deadline farm → Solve skeleton → Export FBX.
  4. Publish: Auto-submit to Perforce via P4Python → Notify artist via Slack webhook.

Python: The File Watcher (Sweeper)

from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler
import time
import subprocess

class CaptureIngestHandler(FileSystemEventHandler):
    """Watches a directory for new .c3d files and auto-submits to farm."""
    
    VALID_EXTENSIONS = {'.c3d', '.fbx', '.bvh'}
    
    def on_created(self, event):
        if event.is_directory:
            return
        
        ext = os.path.splitext(event.src_path)[1].lower()
        if ext not in self.VALID_EXTENSIONS:
            return
        
        # Wait for file to finish writing (network copy)
        self._wait_for_stable(event.src_path)
        
        # Validate
        if not self._validate(event.src_path):
            log.error(f"Validation FAILED: {event.src_path}")
            return
        
        # Submit to Farm
        log.info(f"New capture detected: {event.src_path}")
        self._submit_to_deadline(event.src_path)
    
    def _wait_for_stable(self, path, interval=2, checks=3):
        """Wait until file size stops changing (copy complete)."""
        sizes = []
        for _ in range(checks):
            sizes.append(os.path.getsize(path))
            time.sleep(interval)
        return len(set(sizes)) == 1  # All sizes identical = stable
    
    def _validate(self, path):
        size_mb = os.path.getsize(path) / (1024**2)
        if size_mb < 1:
            return False  # Suspiciously small
        if size_mb > 50000:
            return False  # >50GB? Something is wrong.
        return True
    
    def _submit_to_deadline(self, path):
        subprocess.Popen([
            "deadlinecommand", 
            "-SubmitCommandLineJob",
            "-executable", "mayapy",
            "-arguments", f"-batch -script solve.py --input {path}",
            "-name", f"AutoSolve_{os.path.basename(path)}"
        ])

# Usage: Run as a daemon/service
observer = Observer()
observer.schedule(CaptureIngestHandler(), "/mnt/nas/capture_inbox/", recursive=True)
observer.start()

3. Vendor API Integration

EA works with vendors like Vicon, OptiTrack, Faceware, DI4D, Move.ai, etc. Their data formats are proprietary. Your job is to normalize them.

🧠 The Adapter Pattern

Never let a vendor's format infect your pipeline. Write an Adapter that converts vendor data into YOUR internal format. If the vendor changes their SDK from v3 to v4, you rewrite ONE file (the adapter), not 50 tools.

Python: The Vendor Adapter (Full Implementation)

from abc import ABC, abstractmethod
import json

class CaptureData:
    """Our internal, vendor-agnostic data format."""
    def __init__(self):
        self.markers = []        # List[List[Vector3]] per frame
        self.marker_names = []   # List[str]
        self.frame_rate = 120
        self.take_name = ""
        self.frame_count = 0
        self.metadata = {}       # Arbitrary key-value pairs

    def to_json(self, filepath):
        """Serialize to our internal JSON manifest."""
        with open(filepath, 'w') as f:
            json.dump({
                'take_name': self.take_name,
                'frame_rate': self.frame_rate,
                'frame_count': self.frame_count,
                'marker_count': len(self.marker_names),
                'marker_names': self.marker_names,
                'metadata': self.metadata
            }, f, indent=2)

class BaseCaptureAdapter(ABC):
    """Abstract base class. All vendor adapters must implement load()."""
    @abstractmethod
    def load(self, filepath) -> CaptureData:
        pass

class ViconAdapter(BaseCaptureAdapter):
    """Converts Vicon .c3d files into our internal format."""
    def load(self, filepath):
        import c3d
        reader = c3d.Reader(open(filepath, 'rb'))
        
        data = CaptureData()
        data.frame_rate = reader.header.frame_rate
        data.take_name = reader.header.events[0].desc if reader.header.events else ""
        data.marker_names = reader.point_labels
        
        for frame_idx, points, analog in reader.read_frames():
            data.markers.append(points[:, :3].tolist())
            data.frame_count += 1
        
        return data

class OptiTrackAdapter(BaseCaptureAdapter):
    """Converts OptiTrack .csv exports into our internal format."""
    def load(self, filepath):
        import csv
        data = CaptureData()
        with open(filepath) as f:
            reader = csv.DictReader(f, delimiter=',')
            for row in reader:
                # Parse OptiTrack's specific column layout
                data.markers.append([float(row['X']), float(row['Y']), float(row['Z'])])
                data.frame_count += 1
        return data

class MoveAIAdapter(BaseCaptureAdapter):
    """Converts Move.ai JSON exports (markerless mocap)."""
    def load(self, filepath):
        data = CaptureData()
        with open(filepath) as f:
            raw = json.load(f)
        data.frame_rate = raw.get('fps', 30)
        data.take_name = raw.get('name', '')
        data.markers = raw.get('joints', [])
        data.frame_count = len(data.markers)
        return data

# === Factory Pattern: Auto-detect vendor by extension ===
ADAPTER_MAP = {
    '.c3d':  ViconAdapter,
    '.csv':  OptiTrackAdapter,
    '.json': MoveAIAdapter,
}

def load_capture(filepath) -> CaptureData:
    """Factory: automatically picks the right adapter."""
    ext = os.path.splitext(filepath)[1].lower()
    adapter_class = ADAPTER_MAP.get(ext)
    if not adapter_class:
        raise ValueError(f"Unsupported capture format: {ext}")
    return adapter_class().load(filepath)

4. System Reliability: The Maya Update Problem

"What happens when Maya 2025 ships and breaks your tools?"

  • Pinning: Never use "latest". Pin your Maya version in Docker: FROM maya:2024.2.
  • Abstraction Layer: Wrap maya.cmds in YOUR API. If Maya changes cmds.parent(), you fix ONE wrapper file.
  • CI Gate: Run your test suite against Maya 2025 in a separate CI branch before rolling it out to artists.
  • Rollback Plan: Keep Maya 2024 Docker images alive. If 2025 fails, revert the farm in minutes.

import maya.cmds as cmds
import logging

log = logging.getLogger("dcc_bridge")

def get_selected():
    """Returns selected nodes as long paths."""
    return cmds.ls(selection=True, long=True) or []

def parent_node(child, parent):
    """Wraps cmds.parent with error handling and logging."""
    try:
        cmds.parent(child, parent)
        log.debug(f"Parented {child} -> {parent}")
    except RuntimeError as e:
        log.error(f"Parent failed: {child} -> {parent}: {e}")
        raise

def export_fbx(filepath, selected_only=True):
    """Wraps FBX export with studio defaults."""
    cmds.loadPlugin("fbxmaya", quiet=True)
    
    # Studio-standard FBX settings
    mel.eval('FBXExportSmoothingGroups -v true')
    mel.eval('FBXExportInputConnections -v false')
    mel.eval('FBXExportBakeComplexAnimation -v true')
    
    if selected_only:
        cmds.file(filepath, force=True, type="FBX export", exportSelected=True)
    else:
        cmds.file(filepath, force=True, type="FBX export", exportAll=True)
    
    log.info(f"FBX Export: {filepath}")

def get_skeleton_hierarchy(root_joint):
    """Returns the full joint chain from a root."""
    joints = cmds.listRelatives(root_joint, allDescendents=True, type="joint") or []
    return [root_joint] + joints

def validate_skeleton(root_joint, expected_joints):
    """Check if skeleton matches the studio template."""
    actual = set(cmds.ls(cmds.listRelatives(root_joint, ad=True, type="joint"), shortNames=True))
    expected = set(expected_joints)
    missing = expected - actual
    extra = actual - expected
    if missing:
        log.error(f"Missing joints: {missing}")
    if extra:
        log.warning(f"Extra joints: {extra}")
    return len(missing) == 0

Python: Context Manager for Scene Safety

import contextlib
import maya.cmds as cmds

@contextlib.contextmanager
def undo_chunk(name="PipelineOperation"):
    """Groups all operations into a single Undo.
    If the tool crashes, you Ctrl+Z ONCE to undo everything."""
    cmds.undoInfo(openChunk=True, chunkName=name)
    try:
        yield
    except Exception:
        cmds.undoInfo(closeChunk=True)
        cmds.undo()  # Auto-rollback on crash
        raise
    else:
        cmds.undoInfo(closeChunk=True)

@contextlib.contextmanager
def maintain_selection():
    """Restore the user's selection after the tool runs."""
    original = cmds.ls(selection=True, long=True)
    try:
        yield
    finally:
        if original:
            cmds.select(original)
        else:
            cmds.select(clear=True)

# Usage:
with undo_chunk("BatchExport"), maintain_selection():
    export_all_characters()  # Even if this crashes, scene is safe.

5. Automated Validation: Sanity Checks

"How do you automate quality checks for captured data?"

Python: The Validation Framework

class ValidationResult:
    def __init__(self, name, passed, message=""):
        self.name = name
        self.passed = passed
        self.message = message

class CaptureValidator:
    """Runs a gauntlet of checks on processed capture data."""

    def __init__(self, config):
        self.config = config  # JSON config with thresholds

    def validate_all(self, fbx_path) -> list:
        results = []
        results.append(self.check_file_integrity(fbx_path))
        results.append(self.check_skeleton_hierarchy(fbx_path))
        results.append(self.check_frame_range(fbx_path))
        results.append(self.check_foot_contact(fbx_path))
        results.append(self.check_naming_convention(fbx_path))
        return results

    def check_file_integrity(self, path):
        """Can Maya even open this file?"""
        try:
            cmds.file(path, open=True, force=True)
            return ValidationResult("FileLoad", True)
        except:
            return ValidationResult("FileLoad", False, "File is corrupt or unreadable.")

    def check_skeleton_hierarchy(self, path):
        """Does Root -> Hips -> Spine exist?"""
        expected = self.config['expected_hierarchy']  # ["Root","Hips","Spine",...]
        root = cmds.ls(type="joint")[0]
        for joint_name in expected:
            if not cmds.objExists(joint_name):
                return ValidationResult("Hierarchy", False, f"Missing joint: {joint_name}")
        return ValidationResult("Hierarchy", True)

    def check_frame_range(self, path):
        """Is the animation absurdly short or long?"""
        start = cmds.playbackOptions(q=True, min=True)
        end = cmds.playbackOptions(q=True, max=True)
        duration = end - start
        if duration < 10:
            return ValidationResult("FrameRange", False, f"Only {duration} frames. Likely truncated.")
        if duration > 50000:
            return ValidationResult("FrameRange", False, f"{duration} frames. Suspiciously long.")
        return ValidationResult("FrameRange", True, f"{int(duration)} frames OK.")

    def check_foot_contact(self, path):
        """Detect foot skating: velocity at 'grounded' frames should be ~0."""
        # Simplified: check if foot joints have near-zero velocity
        # when they are below a height threshold (on the floor).
        foot_joints = ["LeftFoot", "RightFoot"]
        for jnt in foot_joints:
            for frame in range(int(cmds.playbackOptions(q=True, min=True)),
                               int(cmds.playbackOptions(q=True, max=True))):
                cmds.currentTime(frame)
                pos = cmds.xform(jnt, q=True, ws=True, t=True)
                if pos[1] < 2.0:  # Below 2cm = "grounded"
                    cmds.currentTime(frame + 1)
                    pos2 = cmds.xform(jnt, q=True, ws=True, t=True)
                    delta = sum((a-b)**2 for a,b in zip(pos, pos2)) ** 0.5
                    if delta > 5.0:  # More than 5cm movement while "grounded"
                        return ValidationResult("FootContact", False,
                            f"Foot skating at frame {frame}: {delta:.1f}cm")
        return ValidationResult("FootContact", True)

    def check_naming_convention(self, path):
        """Filename must match: project_date_shotID_version.fbx"""
        import re
        basename = os.path.basename(path)
        pattern = r'^[a-z]+_\d{8}_sc\d+_sh\d+_v\d{3}\.fbx$'
        if not re.match(pattern, basename):
            return ValidationResult("Naming", False,
                f"'{basename}' doesn't match schema: project_YYYYMMDD_scNN_shNN_vNNN.fbx")
        return ValidationResult("Naming", True)

Python: Cloud Burst to AWS

import boto3

def submit_cloud_job(scene_file, s3_bucket):
    """When local farm is full, burst to AWS Batch."""
    s3 = boto3.client('s3')
    batch = boto3.client('batch')
    
    # 1. Upload scene to S3
    s3_key = f"inbox/{os.path.basename(scene_file)}"
    s3.upload_file(scene_file, s3_bucket, s3_key)
    
    # 2. Submit AWS Batch job (runs our Docker container)
    response = batch.submit_job(
        jobName=f"solve_{os.path.basename(scene_file)}",
        jobQueue="capture-processing-queue",
        jobDefinition="maya-solve-job:3",  # Our Docker image
        containerOverrides={
            'environment': [
                {'name': 'INPUT_S3_KEY', 'value': s3_key},
                {'name': 'OUTPUT_S3_PREFIX', 'value': 'output/'},
            ]
        }
    )
    return response['jobId']

Python: Auto-Publishing Processed Assets

from P4 import P4, P4Exception

def publish_to_perforce(local_path, depot_path):
    """Auto-submit a processed asset to Perforce."""
    p4 = P4()
    p4.user = os.environ['P4USER']
    p4.port = os.environ['P4PORT']
    p4.client = os.environ['P4CLIENT']
    
    try:
        p4.connect()
        
        # Check if file exists in depot
        try:
            p4.run_fstat(depot_path)
            # File exists: edit it
            p4.run_edit(depot_path)
        except P4Exception:
            # File is new: add it
            p4.run_add(local_path)
        
        # Submit with a structured changelist
        change = p4.fetch_change()
        change['Description'] = (
            f"[AUTO] Pipeline publish: {os.path.basename(local_path)}\n"
            f"Source: Capture Farm\n"
            f"Validated: True\n"
            f"Hash: {calculate_md5(local_path)}"
        )
        p4.run_submit(change)
        
    except P4Exception as e:
        log.error(f"P4 Error: {e}")
        raise
    finally:
        p4.disconnect()

8. Bones, Retargeting & Skeletons (Quick Reference)

Fast-reference table if Nigel probes on capture fundamentals.

Term One-Liner
Rigid Body A cluster of 3+ non-collinear markers that define one bone's 6DOF (position + rotation).
Solving Fitting a skeleton inside a point cloud using SVD / Least Squares.
Retargeting Mapping motion from Actor skeleton → Game character (different proportions). Uses HumanIK constraints.
IK "Put the foot HERE." Solver calculates knee/hip angles. Used for contact (feet, hands).
FK "Rotate the shoulder 45°." Child bones follow. Used for free-swinging motion (arms).
Marker Swap Mislabeled marker after occlusion. Detect by velocity spike (impossible speed = error).
Residual Error metric (mm) of camera ray intersection. < 1mm = gold. > 10mm = garbage.
HumanIK (HIK) MotionBuilder's full-body retargeting solver. Maps source to target via characterization.
Characterization Mapping YOUR skeleton's joint names to HIK's internal template. "LeftUpLeg" → your "L_Thigh".
Foot Skating Feet sliding on ground = bad IK solve. Detect by checking velocity at grounded frames.
T-Pose / A-Pose The "Reset" pose. Both solver and retarget need to agree on the rest pose. Mismatch = broken.
.c3d Format Industry standard for 3D point data. Contains marker positions, analog data, and metadata.

9. Photogrammetry Pipeline

If the role involves 3D scanning (likely for "Future Development"), know this flow:

┌─────────────────────────────────────────────────────────────────┐ │ PHOTOGRAMMETRY PIPELINE │ ├─────────────────────────────────────────────────────────────────┤ │ │ │ [200 Photos] → [Alignment] → [Dense Cloud] → [Mesh] → [Texture]│ │ (DSLR/LED) (SfM) (MVS) (Poisson) (UV) │ │ │ │ Tool: RealityCapture / Metashape / Meshroom (open source) │ │ │ │ Output: High-poly mesh (50M tris) → Decimate → Retopo → LOD │ │ Final: Game-ready mesh (10K tris) + Normal Map + Diffuse │ │ │ └─────────────────────────────────────────────────────────────────┘
  • SfM (Structure from Motion): Computes camera positions from overlapping photos.

  • LOD (Level of Detail): Multiple mesh resolutions: 50K (cinematic), 10K (gameplay), 2K (background).
  • Automation: RealityCapture has a CLI: RealityCaptureApp -set "Input Folder" photos/ -align -reconstruct -exportModel output.obj

11. The Infrastructure Bottleneck Story

🎬 Your Story: Solving a Pipeline Bottleneck

Problem: "Our capture team was producing 200 takes per day, but our processing pipeline could only handle 50. Artists were waiting 3-4 days for cleaned data. The bottleneck was a single workstation running MotionBuilder sequentially."

Diagnosis: "I profiled the process. Each take took 12 minutes: 2 minutes to solve, 10 minutes waiting for file I/O on a shared network drive. The real bottleneck wasn't CPU—it was the NAS throughput."

Solution: "Three changes: (1) I containerized the solve into a headless Docker image and deployed it to 20 Deadline farm nodes. (2) I wrote a Python file watcher (`watchdog`) that auto-detected new .c3d files and submitted farm jobs. (3) I moved the intermediate cache to each node's local SSD, only writing the final .fbx to the NAS. This eliminated the I/O bottleneck."

Result: "Processing went from 50 takes/day to 200+/day. Overnight turnaround. Captured Monday → game-ready Tuesday morning. No manual intervention. I also added MD5 validation and Slack notifications."

Key architecture: "Decoupling. The capture stage writes to a buffer (NAS inbox). The sweeper reads from the buffer. The farm processes using local disks. Perforce publishes the final output. No single point of failure."

12. Global Collaboration: The Multi-Site Question

"If a capture happens in Vancouver, how does Stockholm use it?"

  1. Perforce Proxy: A caching server in Stockholm. First request is slow (WAN). Every request after is instant (cached).
  2. Pre-fetch Script: Runs at 3AM Stockholm time. Force-syncs yesterday's dailies so data is local when artists arrive at 9AM.
  3. Tiered Replication: Raw data stays in Vancouver (Cold). Game-ready FBX is replicated to all sites (Hot).
  4. Metadata First: Publish a small JSON manifest immediately (take name, frame count, thumbnail). Artists can search/preview before syncing the heavy 2GB file.
  5. Bandwidth Budget: Schedule large syncs for off-peak hours. Use p4 sync --parallel=threads=8 for throughput.

  • "How do you handle global collaboration? If a capture happens in Vancouver, how is that data optimized for Stockholm?"
  • "What does the team's relationship with external vendors (Vicon, DI4D) look like in terms of API integration?"
  • "What is the team's stance on markerless mocap (Move.ai, DeepMotion)? Is that on the roadmap?"

Key Takeaways

  • Think Systems, Not Scripts: Every answer should mention reliability, scalability, and automation.
  • Adapter Pattern: Isolate vendor dependencies behind a clean internal API. Use a Factory to auto-detect formats.
  • IaC: Your pipeline should be rebuildable from git clone + docker build.
  • CLI-First: Every tool needs a command-line interface for automation and farm usage.
  • Overnight Turnaround: Captured Monday → game-ready Tuesday. That's the benchmark.
  • Pin Everything: Maya version, Python version, library versions. Never rely on "latest".
  • Validation is Non-Negotiable: MD5 hashes, skeleton checks, naming conventions—automate all of it.
