The Universal Code Vision: Markerless + Marker Pipeline Unification

The 2–5 year strategic direction: one pipeline that services both marker-based and markerless mocap, without maintaining two codebases.

💡 Why This Lesson Is the Most Important

The stage TA told you this directly: the long-term goal of the studio is to incorporate markerless tracking into the existing pipeline — not as a separate system, but as a unified pipeline that services both. Your predecessor in this role was focused on building "universal code" — generic, high-level abstraction layers that sit above the specifics of what vendor or technology is doing the actual work. This is the strategic heart of the job.

1. Marker-Based vs. Markerless: How the Data Differs

Before unifying the pipelines, understand what makes them different:

Aspect Marker-Based (Current) Markerless (Future)
Input Infrared cameras + reflective markers RGB video cameras (sometimes depth)
Raw Data .c3d (3D point cloud, kilobytes/frame) Video files (.mp4/.mov, megabytes/frame)
Solving Rigid body math (SVD) on marker clusters Deep learning inference (neural network)
Accuracy Sub-millimeter (gold standard) ~5-15mm (improving rapidly)
Setup Time High (marker suit, calibration) Minimal (just cameras)
Solver Software Vicon Shogun, OptiTrack Motive Move.ai, DeepMotion, Radical, MediaPipe
Output Joint rotations + positions per frame Joint rotations + positions per frame
Post-Production Marker swap fixes, contact cleanup Jitter filtering, penetration fixes, contact cleanup
Final Deliverable .fbx with skeleton animation .fbx with skeleton animation
🧠 The Key Insight: Convergence at the Output

The input is completely different. The solving is completely different. But the output is identical: joint rotations and positions per frame, baked onto a skeleton, exported as FBX. This is why unification is possible — the pipelines diverge at the beginning but converge at the middle.

2. The Universal Code Philosophy

Your predecessor's approach — and the approach the studio wants you to continue — is to write code that is agnostic to the capture technology. The code doesn't know or care whether the data came from markers or from a neural network. It only cares about the data format at the convergence point.

3. Design Pattern: The Strategy Pattern

The Strategy Pattern lets you swap the "how" while keeping the "what" constant. The pipeline defines abstract stages. Each stage has multiple implementations (strategies) that can be swapped via configuration:

Python: Abstract Pipeline Stages

"""
The Universal Pipeline — Abstract stages with swappable strategies.
"""
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import List, Dict, Optional
import logging

log = logging.getLogger("universal_pipeline")

# ═══════════════════════════════════════════
# THE CONVERGENCE FORMAT
# ═══════════════════════════════════════════
@dataclass
class CaptureResult:
    """The universal internal format. Both marker and markerless
    pipelines produce this SAME object."""
    
    take_name: str = ""
    frame_rate: float = 120.0
    frame_count: int = 0
    
    # Joint data: dict of joint_name -> list of (position, rotation) per frame
    joints: Dict[str, list] = field(default_factory=dict)
    joint_hierarchy: Dict[str, str] = field(default_factory=dict)  # child -> parent
    
    # Metadata
    source_technology: str = ""   # "marker" or "markerless"
    source_vendor: str = ""       # "vicon", "moveai", etc.
    source_file: str = ""
    confidence: float = 1.0       # Markerless has per-frame confidence
    
    metadata: Dict = field(default_factory=dict)


# ═══════════════════════════════════════════
# ABSTRACT PIPELINE STAGES
# ═══════════════════════════════════════════
class IngestStage(ABC):
    """Stage 1: Read raw data from any source."""
    @abstractmethod
    def ingest(self, input_path) -> CaptureResult:
        pass

class CleanupStage(ABC):
    """Stage 2: Technology-specific cleanup."""
    @abstractmethod
    def cleanup(self, data: CaptureResult) -> CaptureResult:
        pass

class RetargetStage(ABC):
    """Stage 3: Map to target skeleton."""
    @abstractmethod
    def retarget(self, data: CaptureResult, target_skeleton: str) -> CaptureResult:
        pass

class ValidateStage(ABC):
    """Stage 4: Quality assurance checks."""
    @abstractmethod
    def validate(self, data: CaptureResult) -> List[dict]:
        pass

class ExportStage(ABC):
    """Stage 5: Write to output format."""
    @abstractmethod
    def export(self, data: CaptureResult, output_path: str, profile) -> str:
        pass

"""
Markerless implementations of each pipeline stage.
"""

class MarkerlessIngest(IngestStage):
    """Ingest markerless mocap data (Move.ai, MediaPipe, etc.)."""
    
    def ingest(self, input_path) -> CaptureResult:
        import json
        
        with open(input_path) as f:
            raw = json.load(f)
        
        result = CaptureResult()
        result.take_name = raw.get("name", "")
        result.frame_rate = raw.get("fps", 30)
        result.source_technology = "markerless"
        result.source_vendor = raw.get("vendor", "moveai")
        result.source_file = input_path
        
        # Markerless data often has per-frame confidence
        result.confidence = raw.get("average_confidence", 0.85)
        
        # Joint data from ML inference
        for joint_name, frames in raw.get("joints", {}).items():
            result.joints[joint_name] = frames
            result.frame_count = max(result.frame_count, len(frames))
        
        # Markerless typically provides hierarchy too
        result.joint_hierarchy = raw.get("hierarchy", {})
        
        return result


class MarkerlessCleanup(CleanupStage):

4. The Universal Pipeline Runner

The runner assembles the right strategies based on the source technology and runs them in order:

Python: The Universal Runner

"""
Universal Pipeline Runner — Picks the right strategies based on input type
and runs the shared stages identically for both technologies.
"""

class UniversalPipeline:
    """
    The core pipeline. Technology-agnostic after the convergence point.
    """
    
    # Strategy registry: maps technology to implementations
    STRATEGIES = {
        "marker": {
            "ingest": MarkerIngest,
            "cleanup": MarkerCleanup,
        },
        "markerless": {
            "ingest": MarkerlessIngest,
            "cleanup": MarkerlessCleanup,
        }
    }
    
    def __init__(self, client_registry, plugin_manager):
        self.registry = client_registry
        self.plugins = plugin_manager
    
    def process(self, input_path, technology, client_id):
        """
        Process a capture file through the universal pipeline.
        
        Args:
            input_path:  Path to the raw capture data
            technology:  "marker" or "markerless"
            client_id:   Which client to deliver to
        """
        profile = self.registry.get_profile(client_id)
        strategies = self.STRATEGIES[technology]
        
        log.info(f"Processing [{technology}] for [{client_id}]: {input_path}")
        
        # ─── DIVERGENT STAGES (technology-specific) ───
        
        # Stage 1: Ingest (technology-specific)
        ingest = strategies["ingest"]()
        data = ingest.ingest(input_path)
        
        # Stage 2: Cleanup (technology-specific)
        cleanup = strategies["cleanup"]()
        data = cleanup.cleanup(data)
        
        # ─── CONVERGENCE POINT ───
        # From here on, the code is IDENTICAL for both technologies.
        # This is the "universal code."
        
        # Stage 3: Retarget (universal)
        from pipeline.retarget import HumanIKRetarget
        retargeter = HumanIKRetarget()
        data = retargeter.retarget(data, profile.skeleton_template)
        
        # Stage 4: Validate (universal)
        from pipeline.validation import UniversalValidator
        validator = UniversalValidator(profile)
        results = validator.validate(data)
        
        failures = [r for r in results if not r["passed"]]
        if failures:
            log.error(f"Validation failures: {failures}")
            return False
        
        # Stage 5: Export (universal, format from client config)
        from pipeline.export import get_export_adapter
        exporter = get_export_adapter(profile)
        output_path = self._build_output_path(input_path, profile)
        exporter.export(data, output_path, profile)
        
        # Stage 6: Deliver (universal, destination from client config)
        from pipeline.delivery import get_delivery_adapter
        delivery = get_delivery_adapter(profile)
        delivery.deliver(output_path, profile)
        
        log.info(f"✅ Complete: {output_path}")
        return True
    
    def _detect_technology(self, input_path):
        """Auto-detect whether input is marker or markerless."""
        ext = os.path.splitext(input_path)[1].lower()
        if ext in ('.c3d', '.fbx'):
            return "marker"
        elif ext in ('.json', '.mp4', '.mov'):
            return "markerless"
        else:
            raise ValueError(f"Cannot detect technology from: {ext}")

5. The Phased Migration Roadmap

You don't unify pipelines overnight. Here's a practical multi-year roadmap:

Phase Timeline Goal Key Work
Phase 1: Foundation Months 1-6 Define the convergence format Design CaptureResult data class. Build marker ingest + cleanup. Ensure all existing marker pipeline flows through the new format. No behavior change for users — just a refactor under the hood.
Phase 2: Parallel Run Months 6-12 Add markerless as a parallel path Build markerless ingest + cleanup. Run both pipelines on the same test captures. Compare outputs. Measure accuracy delta. Do not ship markerless to production yet.
Phase 3: Validation Parity Months 12-18 Prove markerless quality is sufficient Build A/B comparison tooling. Run side-by-side: same performance, processed through marker and markerless, compared frame-by-frame. Define acceptance criteria (e.g., <3mm average error vs marker).
Phase 4: Canary Production Months 18-24 Ship markerless for low-risk takes Use markerless for background characters, crowd shots, non-hero animations. Keep markers for hero performances. Gather production feedback.
Phase 5: Unified Production Months 24-36 Single pipeline for all captures Markerless accuracy meets bar. Remove marker-specific code paths as they become redundant. Pipeline auto-detects input type and processes accordingly.
🎬 Interview Relevance

If asked "how would you approach incorporating markerless into the existing pipeline?" — walk them through this phased roadmap. It shows you think in terms of risk management (don't ship unproven tech to hero content), graduated rollout (canary before full deployment), and measurable criteria (3mm accuracy threshold).

6. Building the Predecessor's Vision

The previous Senior Pipeline TA was focused on "universal code" — generic abstractions that sit above vendor specifics. Here's how to build on that philosophy:

The Three Principles of Universal Code

  1. The Convergence Principle: Define a clear internal data format (CaptureResult) that is the handoff point between technology-specific code and universal code. Everything below the convergence point is generic.
  2. The Substitution Principle: Any ingest or cleanup strategy can be swapped out without changing the downstream pipeline. If a new markerless vendor appears in 2028, you write ONE new adapter. The 50 downstream tools don't change.
  3. The Configuration Principle: The choice of technology ("marker" vs "markerless") is a config parameter, not a code branch. The pipeline reads the config and assembles the right strategies. No if technology == "marker" in the core.

7. A/B Comparison Tooling

During Phase 3, you need to prove markerless quality. Here's a conceptual A/B comparison tool:

Python: Marker vs Markerless Quality Comparison

"""
A/B Comparison Tool — Quantifies the quality delta between marker and 
markerless processing of the same performance.
"""
import numpy as np
import json
import logging

log = logging.getLogger("ab_compare")

class CaptureComparator:
    """Compare two CaptureResult objects (same performance, different technology)."""
    
    def compare(self, marker_result, markerless_result):
        """
        Returns a quality report comparing the two results.
        """
        report = {
            "take_name": marker_result.take_name,
            "frame_count": marker_result.frame_count,
            "joint_errors": {},
            "global_metrics": {},
        }
        
        # Per-joint error analysis
        for joint_name in marker_result.joints:
            if joint_name not in markerless_result.joints:
                report["joint_errors"][joint_name] = {"status": "MISSING_IN_MARKERLESS"}
                continue
            
            marker_frames = np.array(marker_result.joints[joint_name])
            markerless_frames = np.array(markerless_result.joints[joint_name])
            
            # Truncate to same length
            min_frames = min(len(marker_frames), len(markerless_frames))
            marker_frames = marker_frames[:min_frames]
            markerless_frames = markerless_frames[:min_frames]
            
            # Position error (Euclidean distance per frame)
            position_errors = np.linalg.norm(
                marker_frames[:, :3] - markerless_frames[:, :3], axis=1
            )
            
            report["joint_errors"][joint_name] = {
                "mean_error_mm": float(np.mean(position_errors)),
                "max_error_mm": float(np.max(position_errors)),
                "p95_error_mm": float(np.percentile(position_errors, 95)),
                "worst_frame": int(np.argmax(position_errors)),
            }
        
        # Global metrics
        all_errors = [
            v["mean_error_mm"] for v in report["joint_errors"].values()
            if isinstance(v, dict) and "mean_error_mm" in v
        ]
        
        report["global_metrics"] = {
            "average_joint_error_mm": float(np.mean(all_errors)),
            "worst_joint": max(report["joint_errors"].keys(),
                key=lambda k: report["joint_errors"][k].get("mean_error_mm", 0)),
            "pass": float(np.mean(all_errors)) < 3.0,  # 3mm threshold
        }
        
        log.info(
            f"A/B Compare: avg error = {report['global_metrics']['average_joint_error_mm']:.2f}mm, "
            f"PASS = {report['global_metrics']['pass']}"
        )
        
        return report

Key Takeaways

  • Different input, same output: Marker-based and markerless capture produce completely different raw data, but the final output is identical: joint rotations on a skeleton, exported as FBX.
  • The Convergence Point: Define a universal internal format (CaptureResult) where both technologies meet. Everything below this point is shared code.
  • Strategy Pattern: Swap ingest and cleanup implementations via config. The core pipeline doesn't know which technology is running.
  • Phased migration over 2–3 years: Foundation → Parallel run → Validation parity → Canary production → Unified production. Never ship unproven tech to hero content.
  • Universal code = abstraction above vendor specifics. If a new vendor appears in 2028, you write one adapter. The 50 downstream tools don't change.
  • Frame beginner experience as fresh perspective. You see the architecture clearly because you're not biased by "how it's always been done."
